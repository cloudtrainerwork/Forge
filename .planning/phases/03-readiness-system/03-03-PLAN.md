## Goal

Implement screen-based grouping and sprint planning features to organize deliverables by screens/features with timeline tracking, supporting the workflow patterns shown in the screenshots.

## Context

From user requirements and screenshots:
- Break screens into multiple component nodes (services, DTOs, tests, etc.)
- Group nodes by screen/feature for organized tracking
- Support sprint planning with timeline (June-August example)
- Red borders indicate "on the bubble" items that may not make release
- Architects use this for sprint planning to ensure nothing is missed
- Support hierarchical workflows (main process → sub-processes → tasks)

## Tasks

Create domain entities for screen grouping

1. Create src/domain/entities/ScreenGroup.ts:
   - Define ScreenGroup with id, name, description
   - Add nodeIds array for grouped work items
   - Include aggregated readiness calculation
   - Add "onTheBubble" flag for at-risk items
   - Support parent-child group relationships
   - Add color coding configuration

   - Add plannedGroups array for screen groups
   - Include velocity/capacity planning fields

</task>

<task name="create-grouping-service">

1. Create src/services/GroupingService.ts:

   - markGroupOnBubble: Flag at-risk groups
   - getGroupHierarchy: Get parent-child structure
   - breakScreenIntoComponents: Auto-create component nodes

2. Add helper methods:
   - suggestComponentsForScreen: AI-assisted breakdown
   - validateGroupCompleteness: Check for missing components
   - identifyMissingDependencies: Find gaps in planning
   - calculateReleaseRisk: Assess bubble probability
</task>

<task name="create-api-endpoints">
Add REST API for grouping and sprints

1. Create src/api/routes/grouping.ts:
   - POST /groups - Create screen group
   - PUT /groups/:id/nodes - Update group nodes
   - GET /groups/:id/readiness - Get aggregated readiness
   - POST /groups/:id/break-down - Auto-create components
   - PUT /groups/:id/bubble - Mark as at-risk

2. Create src/api/routes/sprints.ts:
   - POST /sprints - Create sprint
   - PUT /sprints/:id/groups - Assign groups
   - GET /sprints/:id/timeline - Get timeline data

   - POST /sprints/auto-plan - Auto-distribute work
</task>
