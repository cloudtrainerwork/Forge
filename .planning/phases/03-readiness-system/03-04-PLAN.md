## Goal

Build comprehensive reporting and analytics capabilities to track readiness across screens, sprints, and the entire project, with export functionality and critical path analysis.

## Context

From user requirements:
- Workflow-level readiness summaries (e.g., "Loan Origination is 60% ready")
- Critical path analysis based on readiness states
- Time-to-completion estimates based on velocity
- Sprint-level aggregated readiness
- Identify blockers and at-risk items
- Support for all types of reports
- Future GitHub Actions integration point

## Tasks

<task name="create-reporting-service">
Build backend reporting service

1. Create src/services/ReportingService.ts:
   - generateReadinessReport: Full project status
   - getScreenReadinessReport: Per-screen breakdown
   - getSprintReadinessReport: Sprint progress
   - getCriticalPathReport: Blocking dependencies
   - getVelocityReport: Historical completion rates
   - getAtRiskReport: "On the bubble" items
   - generateExecutiveSummary: High-level overview

2. Add report generation methods:
   - aggregateByDimension: Readiness by dimension
   - aggregateByGroup: Readiness by screen group
   - aggregateBySprint: Readiness by sprint
   - calculateCompletionPercentage: Overall progress
   - identifyBottlenecks: Slowest moving items
   - projectCompletionDate: Based on velocity

3. Add export formats:
   - exportToJSON: Machine-readable format
   - exportToCSV: Spreadsheet format
   - exportToPDF: Executive reports
   - exportToMarkdown: Documentation format
</task>
