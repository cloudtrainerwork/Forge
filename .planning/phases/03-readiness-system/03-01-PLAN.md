## Goal

Implement backend API for 6-dimensional readiness tracking with configurable states, supporting both percentage-based and multi-state tracking for screen-based deliverables.

## Context

From Phase 3 research and user requirements:
- Every deliverable (screen/component) needs 6 dimensions: Requirements, Design, Frontend, Backend, Integration, Test
- Support both 3-state (NOT_STARTED, IN_PROGRESS, COMPLETE) and percentage-based progress
- Configurable states to allow teams to define custom readiness levels
- Bulk operations for updating multiple nodes efficiently
- Validation rules to enforce dependencies between dimensions

<task name="implement-readiness-api">
Create REST API endpoints for readiness tracking

1. Create src/controllers/ReadinessController.ts:
   - PUT /work-items/:id/readiness - Update single dimension
   - PUT /work-items/readiness/bulk - Bulk updates
   - GET /work-items/:id/readiness/summary - Get aggregated readiness
   - GET /work-items/readiness/filter - Query by readiness state
   - POST /readiness/configuration - Create custom configuration
   - GET /readiness/configuration/:id - Get configuration

2. Create src/api/routes/readiness.ts:
   - Define OpenAPI schemas for requests/responses
   - Add validation middleware for state transitions
   - Include error handling for invalid updates
   - Support both percentage and state-based updates

3. Update src/api/server.ts:
   - Mount readiness routes under /api/v1
   - Add readiness service to dependency injection
</task>

<task name="add-validation-rules">
Implement business rule validation for readiness dependencies
